# RUNBOOK

## Use When
the user asks which files need updates, requests rename/refactor impact analysis, asks for broader touch points, or needs a distinction between direct edits, impacted consumers, generators, and non-issues.

## Workflow
1. Define task scope explicitly.\n2. Identify source-of-truth and transforms.\n3. Identify direct consumers and downstream surfaces.\n4. Separate direct edits, impacted consumers, generator/source implications, verified non-issues, and uncertainty.\n5. Reconcile static findings with runtime/rendered evidence where possible.\n6. Run contradiction check before concluding.
